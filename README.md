# lab-portal-i

本仓库用于归档和发布多个独立 HTML 页面。这些页面通常为静态内容，不依赖特定后端服务，可直接在浏览器中打开或通过静态托管服务访问。

## 项目简介

`lab-portal-i` 是一个轻量级的 HTML 页面集合仓库。其主要目的是集中存放和版本管理各类独立的 HTML 文件，方便后续的查阅、分享或部署。仓库本身不绑定任何特定域名或网站，仅作为页面的归档与发布载体。

## 目录说明

```
lab-portal-i/
├── pages/          # 存放所有 HTML 页面文件的目录
├── assets/         # 页面引用的静态资源（如 CSS、JS、图片等）
├── docs/           # 其他相关文档或说明
└── README.md       # 本文件
```

- **pages/**：每个独立的 HTML 页面文件放置于此，文件名应具有描述性。
- **assets/**：若页面需要外部样式、脚本或媒体文件，统一放置在该目录下，便于管理。
- **docs/**：可存放额外的说明文档、使用指南或变更记录。

## 页面归档说明

- 所有页面均为静态 HTML，不包含服务器端逻辑。
- 页面之间相互独立，无依赖关系。
- 每个页面应尽量自包含，如需引用外部资源，请优先使用相对路径。
- 建议为每个页面添加简单的标题和说明注释，便于他人理解内容。

## 维护说明

- 新增页面：直接在 `pages/` 下创建新的 HTML 文件，并确保引用资源路径正确。
- 更新页面：修改对应文件后，提交变更并注明修改内容。
- 删除页面：确认页面不再需要后，移除文件并更新相关引用。
- 欢迎提交 Pull Request 来改进或补充页面内容，但请保持风格一致。
